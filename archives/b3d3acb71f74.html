Git中修改文件名大小写导致文件丢失

2019-09-09 工具应用

解决"修改文件名大小写"造成的git上传文件丢失

法一

  1. 让 git 区分大小写：
git config core.ignorecase false
  1. 删除缓存：
git rm -r --cached
  1. 剩余照常
git add .
git commit -m 'xxx'
git push xxx

法二

修改文件大小写，然后重新提交

a.txt A.txt
